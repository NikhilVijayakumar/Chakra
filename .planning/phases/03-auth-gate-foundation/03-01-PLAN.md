<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| renderer -> preload auth API | Untrusted renderer input crosses into auth login calls and receives a session token back |
| renderer session state -> route guards | Volatile session state decides whether protected routes are reachable |
| login UI -> user-visible error copy | Error text must not leak internal auth or startup details |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | S | src/preload/index.d.ts + LoginContainerOverride.tsx | mitigate | Replace `any`-typed auth result handling with `AuthLoginResult` and only accept `sessionToken` on explicit success before writing to `volatileSessionStore` |
| T-03-02 | I | LoginContainerOverride.tsx | mitigate | Keep failure messaging generic (`Invalid credentials`) and avoid logging token values or auth internals in the login path |
| T-03-03 | E | SplashContainerOverride.tsx + src/renderer/src/main.tsx | mitigate | Keep route access locked behind `volatileSessionStore.hasSession()` and the existing auth guards so direct route access does not bypass login |
</threat_model>
